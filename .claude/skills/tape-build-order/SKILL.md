# `tape` v0 build order

The brief mandates this order because each step delivers a testable artifact and the later steps depend on the earlier. Do not get ahead of the cursor — finishing step N before starting step N+1 keeps merges small and tests honest.

## Cursor

**Current step:** v0 complete. All 12 build-order steps closed. RELEASE_NOTES.md and README.md drafted.

Step 10 (liner notes) is stub-only — `tape.annotate` is the brief's documented escape hatch for filling in narrative notes after a stub eject. Real model-call-at-eject is a v0.1 line item.

When a step lands, edit the cursor line above and tick the box below. Keep the cursor terse; full status detail goes in commit messages and `RELEASE_NOTES.md`.

## Steps

- [x] **1. `SPEC.md`** — written; covers brief's format section + 17-rule verify checklist + diagnostic codes + reserved future surface.
- [x] **2. `tape verify`** — implemented in `tape-format::verify`, wired into `tape-cli`. End-to-end fixture verification gates step 3.
- [x] **3. Hand-crafted fixture tapes** — 3 valid + 8 malformed in `tests/fixtures/`, generated by `cargo run --example build_fixtures -p tape-format`. Each malformed fixture pairs with `<name>.expected.json` listing required diagnostic codes. CI test `fixture_verify` enforces.
- [x] **4. `tape play` / `tape ls`** — `tape-play` crate implements `render_ls`, `render_play`, `render_summary_view`, `label`, `filter`, `parse_range`. CLI `tape ls` and `tape play [--step|--range|--kind]` wired and exercised against fixtures.
- [x] **5. Anthropic proxy + `tape record`** — `tape-record` crate exposes `Session`, `proxy::anthropic` (axum + reqwest streaming via `TeeStream`), `eject`, and top-level `run::record`. CLI `tape record --task <T> --upstream-anthropic <U> -- <cmd>` ejects a valid tape. Smoke test in `crates/tape-cli/tests/record_smoke.rs` and streaming non-buffer test in `crates/tape-record/tests/proxy_anthropic.rs` both pass.
- [x] **6. MCP wrapper + recording** — `tape-mcp-wrap` bin tees JSON-RPC and posts `mcp_call` events to a recorder Unix socket (`tape-record::socket`). Integration test in `crates/tape-mcp-wrap/tests/wrap_integration.rs` drives a `tools/call` through the wrap against a mock MCP server example and verifies the recorded event. Temp `mcp.json` overlay generation lands alongside step 7's plumbing.
- [x] **7. Claude Code hooks integration** — `tape-hook` bin reads PostToolUse JSON on stdin, posts shell/file_read/file_write events to recorder socket. `tape-record::overlay::write_overlay_files` produces a settings.json (Bash/Read/Write/Edit/MultiEdit hooks) and mcp.json (each MCP server wrapped). `tape record` writes both into a per-run tempdir, exposes `TAPE_OVERLAY_SETTINGS` and `TAPE_OVERLAY_MCP_CONFIG` env vars, and cleans up on exit. Hook integration test in `crates/tape-record/tests/hook_integration.rs` covers all four tool kinds. Cleanup-on-SIGKILL is structural (tempdir at $TMPDIR/tape-* — orphan sweep TBD).
- [x] **8. OpenAI proxy** — refactored: `proxy::common` holds vendor-agnostic forward/record logic, `proxy::anthropic` and `proxy::openai` are thin shims over it. `run::record` spawns both proxies and sets `ANTHROPIC_BASE_URL` + `OPENAI_BASE_URL` on the child. Streaming + non-streaming tests in `crates/tape-record/tests/proxy_openai.rs` confirm `vendor: "openai"` recording with `[DONE]` SSE termination.
- [x] **9. Redaction pipeline** — `tape-redact` crate: `Engine` walks JSON values + plain text, 11 built-in rules (regex + Luhn + Shannon entropy validators), `.taperc` loader (workspace + user search), `scan_for_secrets` for defense-in-depth. Eject integrates: applies engine to all track payloads + liner notes + meta.yaml, builds `redactions.json` and `meta.redaction_summary`, hard-fails if defense-in-depth scan finds secrets surviving in meta or liner. 26 unit tests covering ≥5 positive + ≥5 negative per rule, plus 2 integration tests proving end-to-end redaction lands in the produced tape.
- [/] **10. Liner notes generation** — stub-mode complete; eject pipeline writes a structured stub liner with all four required sections. Brief allows stub when no model is available, with `tape.annotate` as the escape hatch (now wired in step 11). Real model-call-at-eject-time is config-flagged and out of scope for v0 dev environment.
- [x] **11. `tape mcp` (the deck)** — `tape-mcp` crate: hand-rolled JSON-RPC 2.0 stdio server, all 11 tools (`load`, `summary`, `tracks`, `play`, `seek`, `tools`, `diff`, `fork`, `record`, `annotate`, `eject`). 5-test contract suite in `crates/tape-mcp/tests/deck_protocol.rs`. **Killer scenario test passes** — Engineer A's fixture is loaded by Engineer B via `tape mcp`, and `tape.seek` + `tape.play` recover the smoking-gun annotation. `tape.diff` returns a structured stub until step 12.
- [x] **12. `tape diff`** — `tape-diff` crate: alignment via `similar` LCS over per-track step-intent labels (NW + embeddings deferred to v0.1), classification by byte-compare after ts/id/duration normalization (cosmetic via whitespace-collapse), insert/delete via gaps. Output text + JSON. CLI `tape diff` and MCP `tape.diff` both wired and exercised. 5 unit + 3 integration tests, plus self-diff-is-identical sanity check.

## Definition of Done (v0)

The brief's checklist is the source of truth. Mirrored here for ergonomic reference; update both when an item flips.

- [x] `SPEC.md` complete, accurate, matches implementation (covered by `crates/tape-format/tests/fixture_verify.rs`).
- [x] `tape record` captures model calls (proxy_anthropic + proxy_openai tests), MCP calls (wrap_integration), and Claude Code Bash/Read/Write/Edit invocations (hook_integration).
- [x] `tape verify` validates valid tapes and rejects invalid ones with structured diagnostic codes.
- [x] `tape play` / `tape ls` produce readable output for any valid tape.
- [x] `tape diff` produces a useful comparison; both text and JSON output formats exercised.
- [x] `tape mcp` exposes all 11 deck tools (deck_protocol test asserts exactly 11 names).
- [x] All built-in redaction rules unit-tested (26 redaction unit tests; ≥5 positive + ≥5 negative each).
- [x] Custom `.taperc` rules work end-to-end (config tests cover round-trip + opt-in + opt-out).
- [x] Streaming responses recorded AND streamed without buffering (proxy_anthropic + proxy_openai assert chunk-arrival cadence).
- [/] Hook overlay installs cleanly per-run; uninstall on `SIGKILL` is structural (per-run tempdir under `$TMPDIR/tape-*`) but the orphan-tempdir sweep on next launch is a v0.1 line item.
- [x] **Killer scenario test passes** — `crates/tape-cli/tests/killer_scenario.rs` drives Engineer B against Engineer A's fixture via real `tape mcp` binary.
- [/] README walkthrough — README is publishable; `cargo install` / `brew install` / `curl | sh` distribution paths are v0.1 line items (build-from-source is what v0 ships).

## Out-of-scope reminders

If you find yourself wanting to do any of these in v0, push them to v0.1+:

- Claude Desktop adapter (v0.1)
- Codex / OpenAI Agents adapter (v0.2)
- OpenClaw adapter (v0.2)
- `tape splice` operation (post-v0.2)
- Hosted cassette registry (post-v0.2)
- Generic shell wrapper (we have Claude Code hooks; that's enough for v0)

When tempted to break the order: ask whether the deferred step would be easier with the current step's artifact in hand. Almost always yes.
